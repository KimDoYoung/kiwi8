# KDaemon의 역활

● kiwi8 목적(자동매매, 스케줄링, LLM 분석)을 고려한 제안:

  ---
  kdaemon 역할 제안

  핵심 역할: 상시 실행 백그라운드 프로세스

  FastAPI는 요청/응답 처리에 집중하고, kdaemon은 "아무도 요청 안 해도 해야 할 일"을 담당.

  ---
  담당 업무

  1. 장 스케줄러
  - 장 시작(09:00) / 종료(15:30) 이벤트 감지
  - 시간 기반 작업 트리거 (ex. 09:00에 전일 정산, 15:35에 잔고 스냅샷 저장)

  2. WebSocket 수명 관리
  - KIWOOM/LS 체결통보 WebSocket 상시 연결 유지
  - 연결 끊기면 자동 재연결

  3. 시세 데이터 수집
  - 관심 종목 현재가 주기적 수집 → stk_cache 저장
  - 전일종가, 거래량 등 일봉 데이터 EOD 수집

  4. LLM 분석 파이프라인
  - 수집된 데이터로 Local LLM 분석 요청 (장 종료 후)
  - 결과 → DB 저장 → 다음날 참조

  5. 자동매매 엔진
  - 조건 감시 루프 (N초마다 조건 체크)
  - 조건 충족 시 주문 실행

  6. 헬스체크 & 알림
  - FastAPI backend 응답 확인
  - 이상 시 로그/알림

  ---
  구조 제안

  kdaemon.py
  ├── main loop (asyncio event loop)
  ├── JobScheduler   ← 시간 기반 작업
  ├── WsKeeper       ← WebSocket 상시 연결
  ├── PriceCollector ← 시세 수집
  ├── AutoTrader     ← 자동매매 조건 감시
  └── LlmPipeline    ← EOD 분석

  ---
  가장 먼저 구현할 가치 있는 것: WsKeeper + JobScheduler — 이 둘이 없으면 체결통보가 불안정하고 스케줄
  작업이 FastAPI 수명에 종속됨.



## 급등주를 찾자
반도체와 로봇 섹터에서 내일 당장 강하게 튈 만한 종목을 찾아내는 조건검색식(HTS/MTS용) 구성 법입니다.

주도 섹터 내에서 급등 직전의 종목들을 잡아내기 위해서는 **'강한 돈의 유입(거래대금)'**, **'단기 매물 소화(눌림목 및 이평선 밀집)'**, 그리고 '시장의 관심도(거래량 회전율)'를 묶어서 필터링해야 합니다.

국내 HTS(키움증권 영웅문 등)에서 바로 구현할 수 있는 구체적인 조건식 설정을 정리해 드립니다.

## 1. 반도체·로봇 급등 유망주 포착 조건식

아래 조건들을 순서대로 넣고 [전체 AND]로 묶어주시면 됩니다. 단, 대상변경에서 우선주, 관리종목, 투자경고/위험, ETF, ETN, 스팩(SPAC)은 반드시 제외해 주세요.

| 번호 | 조건 항목 | 세부 설정 값 | 기획 의도 |
| --- | --- | --- | --- |
| **A** | **주가범위** | 0일전 종가가 `3,000원 이상 ~ 100,000원 이하` | 가벼우면서도 잡주가 아닌 탄탄한 가격대 |
| **B** | **대상 섹터** | 인포락/업종 분류: `반도체` OR `로봇/지능형로봇` | 반도체와 로봇 종목만 필터링 |
| **C** | **거래대금 순위** | `일 거래대금 상위 100위` 이내 (또는 5일 평균 30억 이상) | 시장의 돈이 쏠려 있는 주도주 확인 |
| **D** | **거래량 회전율** | `최근 5일 이내 최고 거래량 회전율 5% 이상` | 최근 일주일 내에 세력 수급이 들어온 흔적 |
| **E** | **이평선 조건** | `주가(종가) > 5일 이동평균선` AND `5일선 > 20일선` | 역배열 잡주를 거르고, 상승 추세인 종목만 선별 |
| **F** | **눌림목 포착** | `0일전 거래량이 1일전 거래량의 50% 이하` | **핵심:** 돈은 들어왔는데(C, D), 오늘은 거래량이 죽으며 숨고르기 중인 상태 |

## 2. 조건식 검색 후 '내일 튈 3개' 고르는 눈 (필터링 룰)

위 조건식을 돌리면 당일 장 마감 기준으로 대략 5~10개 안팎의 종목이 뜹니다. 이 중에서 내일 아침 가장 유망할 3개를 압축하는 최종 스크리닝 방법입니다.

* **1순위: 재료의 유효성 (뉴스 확인)**
* 반도체라면 'HBM/CXL 신기술', '엔비디아/삼성전자 공급망' 관련 뉴스나 공시가 최근 3일 내에 있었는지 확인합니다. 로봇이라면 '대기업 M&A', '정부 규제 완화', '물류/휴머노이드 상용화' 이슈가 엮여 있어야 힘이 오래 갑니다.


* **2순위: 시간외 단일가 동향**
* 조건식에 걸린 종목 중, 오후 4시~6시 **시간외 단일가 매매에서 거래량이 터지며 2~4% 이상 상승 마감**한 종목이 있다면 내일 아침 시초가 갭상승 후 급등할 확률이 매우 높습니다.


* **3순위: 차트상 '직전 고점'과의 거리**
* 매물대 차트를 켰을 때, 바로 위에 큰 매물벽이 없는 종목이 가볍게 날아갑니다. 직전 매물대를 오늘 살짝 돌파해 놓았거나, 돌파 바로 직전(턱밑)에서 마감한 종목이 베스트입니다.



---

> **Peer's Note**
> 장이 열리기 전 아침 8시 40분부터 9시 사이의 **'예상 체결가 창'**도 꼭 확인하세요. 조건식에 나온 종목 중 아침에 예상 거래량이 실리면서 +3% 이상 갭을 띄워 출발하는 종목이 진짜 대장주입니다. 다만 시초가가 +10% 이상 너무 높게 뜬다면 쫓아가지 않는 것이 안전합니다.

조건검색식을 실제로 HTS에 등록하는 과정에서 특정 조건(예: 특정 이평선 설정이나 정배열 기준 등)을 다듬는 데 어려움이 있으시다면 편하게 말씀해 주세요.

조건검색식의 기본 개념과 시장의 돈이 몰리는 주도주를 포착하기 위해 거래대금 순위를 활용하는 구체적인 설정 방법은 [키움증권 조건검색식 기초 활용법](https://www.youtube.com/watch?v=bbwhFWdbYyw) 영상에서 더 명확하게 확인하실 수 있습니다. 이 영상은 HTS 내에서 수급이 강한 종목들을 스크리닝할 때 필요한 필수 조건들을 차근차근 짚어주어 수급 매매 기준을 잡는 데 도움이 됩니다.


## 나의 도박

1. 3:10 에 **내일 급등할 종목 3개를 고른다** 
2. 각각 1천만원씩으로 매수한다.
3. 그 다음날 장시작하면 3개의 주식에 대해서 가격을 모니터링한다. 계속 기준가를 모니터링한다.
4. 기준가
    - 100->110->120->115->123
    - 위와 같이 오를 때 기준가는 각각
    - 100->110->120->120->123 과 같이 변화한다.
    - 기준가에 -5% 이상이 되면 매도한다.
5. 기준가와 상관없이 3:00에 모두 매도한다. 


위험을 감수하면서 **도파민도 챙기고, 시장의 매운맛도 짜릿하게 느끼되, '나의 도박'보다 수학적 기댓값(승률과 손익비)을 훨씬 더 유리하게 가져가는 방법**이 있습니다.

현재 제안하신 '종가 매수 $\rightarrow$ 시초가 대응'은 밤사이 미 증시라는 컨트롤 불가능한 도박판에 돈을 던져놓고 기도해야 해서 무력감이 큽니다. 진짜 도파민은 내 눈앞에서 호가창이 불타오를 때 내 판단으로 판을 지배하는 데서 오죠.

위험(High Risk)을 즐기면서도 생존 확률을 극대화하는 **3가지 매운맛 트레이딩 전략**을 제안합니다.

## 1. 아침 9시의 야수: '시초가 돌파 돌격' (수익비 극대화)

3시 10분에 사서 밤새 불안해하지 말고, 아침 9시 정각에 시장에서 가장 뜨거운 종목에 바로 올라타는 전략입니다.

* **진입 조건:** 아침 9:00 ~ 9:05 사이, 거래대금이 미친 듯이 터지며 전일 종가 대비 +3%~+5% 수준에서 양봉으로 치고 올라가는 반도체/로봇 대장주를 포착합니다.
* **도파민 포인트:** 장 시작 직후 10분간의 변동성은 롤러코스터 급입니다. 1분 만에 +5%가 찍히는 짜릿함을 실시간으로 맛볼 수 있습니다.
* **나의 도박보다 좋은 이유:** 전일 종가 매수는 '악재 갭하락'을 고스란히 맞지만, 이 방식은 이미 아침에 강한 수급과 호재가 확인된 '달리는 말'에 올라타기 때문에 시초가 폭탄을 피할 수 있습니다.

## 2. 세력의 낙하산 잡기: '장중 투매 급락 유도' (역발상 도박)

호재로 급등하던 대장주가 차익실현 물량이나 세력의 흔들기로 인해 **장중 순식간에 -7% ~ -10% 이상 수직 낙하할 때 투매를 받아내는 전략**입니다.

* **진입 조건:** 당일 주도주(분봉상 정배열로 잘 가던 녀석)가 뚜렷한 악재 없이 프로그램 매도나 순간적인 투매로 지지선까지 급락할 때 분할 매수.
* **도파민 포인트:** 모두가 공포에 질려 던질 때 칼날을 잡는 쾌감이 있습니다. 대장주는 죽더라도 한 번에 안 죽고 강한 반등(기술적 반등)을 주기 때문에, 잡자마자 +3%~5% 반등이 순식간에 나옵니다.
* **나의 도박보다 좋은 이유:** 고점에서 사서 털리는 게 아니라, 남들이 다 털린 바닥(언더슈팅)에서 사기 때문에 손절선이 매우 짧고 명확해집니다.

## 3. 종가 베팅의 업그레이드: '시간외 단일가 스나이퍼'

오후 3시 10분에 무작정 고르는 것이 아니라, 오후 4시부터 6시까지 열리는 **시간외 단일가 시장**을 이용하는 방법입니다.

* **진입 조건:** 3시 30분 장이 끝나고 호재 공시가 뜨거나, 오후 4시 이후 시간외 거래에서 거래량이 평소보다 몇십 배 터지며 +2~3% 상승을 시작하는 종목을 4~5시 사이에 매수합니다.
* **도파민 포인트:** 남들보다 먼저 호재를 포착해 시간외 시장에서 주식을 선점하고, 내일 아침 갭상승을 확정 지은 채 밤을 보낼 수 있습니다.
* **나의 도박보다 좋은 이유:** 3시 10분 매수는 '내일 오를까?' 확신이 없지만, 시간외에서 돈이 들어오는 것을 눈으로 확인하고 사기 때문에 승률이 압도적으로 올라갑니다.

## 💡 도파민을 유지하되 계좌는 살리는 '베팅 룰' 변경

만약 이 "도파민 트레이딩"을 하실 거라면, 기존 룰에서 딱 하나만 바꿔주세요.

> **[기존]** 최고가 대비 -5% 터치 시 매도 (익절은 짧고, 손절은 길어질 수 있음)
> **[변경]** **익절은 자유(예: +7%에 반절 매도), 손절은 진입가 대비 칼같이 -3% 제한**

'나의 도박' 룰의 치명적인 약점은 흔들기 한 번에 내 익절가는 -5% 깎여서 강제 청산되는데, 만약 아침에 시작하자마자 마이너스로 꽂히면 손절 기준이 모호하다는 점이었습니다.

"먹을 때는 길게 먹고, 틀렸을 때는 내 베팅금의 3%만 칩으증권사 1개만을 대상으로 하려고 함. kiwoom 로 지불하고 미련 없이 판을 나온다"는 하우스 룰을 세우시면, 짜릿함은 그대로 가져가면서 타짜처럼 오래 살아남으실 수 있습니다.

가상 시뮬레이션 판돈으로 오늘 당장 '아침 돌격'이나 '시간외 저격' 대상이 될 만한 후보가 보이는지 호가창을 째려보는 것부터 시작해 보시는 건 어떨까요?


프로그램(Daemon)이 자동으로 돌아가는 알고리즘 매매 환경이 기본 베이스라면, **"하루 단위(데이 트레이딩)보다 기간을 조금 더 가져가는 전략(스윙 트레이딩, 보통 2일~2주일 보유)"이 시스템적으로나 기댓값 측면에서 훨씬 유리하고 강력합니다.**

호가창을 보지 않고 데몬을 돌릴 때, 보유 기간을 며칠 더 늘리면 도파민의 결(결과를 확인하는 짜릿함)이 달라지면서도 시스템의 취약점이 대폭 보완됩니다. 왜 보유 기간을 늘리는 게 더 나은지, 그리고 데몬에 탑재하기 좋은 '업그레이드된 도박 룰'을 정리해 드립니다.

## 1. 데몬 자동매매에서 '보유 기간'을 늘려야 하는 이유

### ① 슬리피지와 '페이크(노이즈)' 극복

분 단위, 하루 단위 주가 움직임에는 세력의 허수아비 주문이나 프로그램 매매 꼬임 같은 '노이즈'가 너무 많습니다. 데몬이 최고가 대비 -5%를 감시할 때, 장중 3분 만에 주가를 -6% 밀었다가 종가를 +20%로 마감하는 일명 **'개미 털기'에 알고리즘이 무참히 갈려 나갑니다.** 보유 기간을 며칠로 늘리면 이런 장중 노이즈를 버텨내고 큰 추세를 먹을 수 있습니다.

### ② 거래 비용(세금/수수료) 절감

매일 3,000만 원씩 회전시키면 한 달(20영업일) 누적 거래대금이 6억 원입니다. 거래세와 수수료로만 매달 최소 100만~120만 원이 자동으로 하우스(정부와 증권사)에 뜯깁니다. 회전율을 낮추고 며칠 가져가면 이 비용이 획기적으로 줄어들어, **판돈을 지키면서 더 큰 배당(수익률)을 노릴 수 있습니다.**

### ③ 직장인/개발자 친화적

장중에 실시간 호가창이나 1분봉을 쪼아가며 감시할 필요가 없습니다. 데몬이 **'하루 한 번 장 마감 직전(15:10)'에만 데이터를 수집해서 판단**하고, 포지션을 잡으면 며칠 묵혀두는 방식이므로 서버 리소스나 예외 처리(네트워크 끊김 등) 리스크도 훨씬 적습니다.

## 2. 데몬용 업그레이드 룰: 'N일 보유 추세 추종'

하루 단위가 아닌, '최대 5일(일주일) 보유'를 기준으로 데몬을 짤 때 도파민과 수익을 모두 잡는 세팅법입니다.

### [Step 1] 매수 조건 (15:10에 딱 한 번 체크)

* **조건:** 앞서 언급한 반도체/로봇 조건검색식에 걸린 종목 중, 당일 종가가 5일 이동평균선 바로 위(눌림목 완 완성)인 종목 3개를 골라 종가 매수.

### [Step 2] 매도 알고리즘 (보유 기간 중 실시간 감시)

기존의 트레일링 스톱(최고가 대비 -5%)을 유지하되, **보유 기간 룰**을 추가합니다.

```
if (현재가 <= 매수가 * 0.96) { 
    // 1. 원금 보호선 (-4% 손절) -> 진입 후 바로 꺾이면 칼같이 탈출
    전량매도(); 
} 
else if (현재가 <= 장중최고가 * 0.93) { 
    // 2. 익절선 상향 -> 크게 먹기 위해 버퍼를 -7%로 확대 (장중 흔들기 방어)
    전량매도(); 
} 
else if (보유기간 == 5일째 && 장마감직전) { 
    // 3. 타임컷 -> 위 조건 안 걸렸어도 5일 지났으면 무조건 청산
    전량매도(); 
}

```

* **포인트:** 고점 대비 매도 기준을 -5%에서 **-7%~-10% 정도로 넓혀주는 대신, 최초 손절선(-4%)을 명확히** 잡는 것입니다. 급등주는 하루에도 +-5%씩 흔들리기 때문에, 버퍼를 조금 넓혀줘야 2~3일 만에 +30%, +50% 날아가는 진짜 '대시세'를 데몬이 끝까지 발라먹을 수 있습니다.

## 💡 결론: 도파민의 주기를 바꾸는 재미

매일 아침 9시 5분에 "오늘 얼마 벌었지?" 하고 확인하는 도파민도 좋지만, "월요일에 데몬이 사둔 반도체 종목이 수요일, 목요일 연속 급등하면서 3일 만에 +25%를 찍고 있네?"를 확인하는 도파민이 복리 효과 덕분에 훨씬 더 짜릿합니다.

알고리즘을 완전히 방치하는 스윙(Swing) 방식으로 데몬을 설계하면, 개발자로서 '내가 짠 코드가 시장의 거친 파도를 며칠 동안 이겨내고 수익을 물어오는' 또 다른 차원의 쾌감을 느끼실 수 있습니다.

우선 시뮬레이션 환경을 구축하실 때, **[당일 매수 $\rightarrow$ 당일 매도]** 버전과 **[당일 매수 $\rightarrow$ 최대 5일 보유]** 버전 두 개의 파이프라인을 동시에 가상으로 돌려보며 로그를 비교해 보시는 건 어떨까요? 어떤 쪽이 형님의 성향과 기댓값에 맞는지 데이터가 바로 말해줄 것입니다.
