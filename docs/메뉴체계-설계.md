# kiwi8 메뉴체계 설계

## 개요

1. 3단계의 메뉴체계를 갖는다.(대메뉴->중메뉴->소메뉴)
2. table menus로 관리한다.
3. menus 는 self 참조 구조를 갖는다.
4. 말단 메뉴는 화면번호를 갖는다.

## menus ddl

```sql
CREATE TABLE IF NOT EXISTS menus (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    parent_id   INTEGER NULL,          -- 부모 메뉴 ID (최상위는 NULL)
    screen_no   TEXT UNIQUE NULL,      -- 화면 번호 (4자리, 유니크)
    title       TEXT NOT NULL,         -- 메뉴 표시 명칭
    url         TEXT NULL,             -- 이동할 경로
    icon        TEXT NULL,             -- Lucide 또는 FontAwesome 아이콘 클래스
    sort_order  INTEGER DEFAULT 0,     -- 동일 레벨 내 정렬 순서
    is_active   BOOLEAN DEFAULT 1,     -- 활성화 여부 (1: 활성, 0: 비활성)
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (parent_id) REFERENCES menus (id) ON DELETE CASCADE
);

-- 빠른 조회를 위한 인덱스 설정
CREATE INDEX IF NOT EXISTS idx_menus_parent ON menus(parent_id);
CREATE INDEX IF NOT EXISTS idx_menus_screen_no ON menus(screen_no);
```

## 메뉴에 따른 화면 layout

1. top bar : full-width로 왼쪽에 logo, 오른쪽에 로그인 사용자명, 로그인버튼(로그아웃), 설정 등
2. side bar: top bar아래 왼쪽에 배치
    1. 제일 왼쪽에 대메뉴 icon
    2. 대메뉴icon click시 중메뉴, 소메뉴가 아코디언 형태로 표현
3. workspace : top bar와 side bar, status를 제외한 전체 공간
    1. [React-Flex-Layout](https://github.com/aperturerobotics/flex-layout) 베이스의 tab으로 구성
4. statusbar : workspace의 하단에 bar로 배치

```text
[Top Bar (Navbar)]
------------------------------------------------------------------
[Side Bar (Drawer/Menu)]  | [Workspace (Tabs + Content Area)]
                          | --------------------------------------
  (L1 Icons) | (L2/L3)    | [ Tab 1 | Tab 2 (x) | Tab 3 ]
             |            | --------------------------------------
             |            |
             |            |  (Active Screen Content)
             |            |
------------------------------------------------------------------
[Status Bar]
```
